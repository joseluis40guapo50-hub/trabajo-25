# Seguimiento del Proyecto mediante Git
## Uso de Git como herramienta de monitoreo del progreso

Este documento registra los cambios más relevantes del proyecto simulado
“Gestión de Tareas” y muestra cómo cada commit se relaciona con una actividad
específica del **modelo híbrido Cascada + Scrum**.

El objetivo es evidenciar que Git no solo sirve para controlar versiones de
código, sino también para **dar seguimiento al avance del proyecto**.

---

## 1. Relación entre Git y la metodología aplicada

En este proyecto:
- **Cascada** se utilizó para las fases iniciales de planificación y diseño.
- **Scrum** se utilizó para el desarrollo iterativo y la revisión del proyecto.

Cada commit representa una **actividad concreta** dentro de estas fases.

---

## 2. Registro de commits y actividades del proyecto

### Commit 1
**Mensaje del commit:**


**Metodología:** Cascada  
**Actividad asociada:** Planificación y análisis  
**Descripción:**  
Se definieron los objetivos del proyecto, el alcance y las funcionalidades
principales. Este commit marca el inicio formal del proyecto.

---

### Commit 2
**Mensaje del commit:**


**Metodología:** Cascada  
**Actividad asociada:** Diseño general  
**Descripción:**  
Se estableció la estructura del repositorio y se documentó el diseño del
proyecto antes de iniciar la implementación.

---

### Commit 3
**Mensaje del commit:**


**Metodología:** Scrum  
**Actividad asociada:** Sprint 1 – Desarrollo  
**Descripción:**  
Se agregaron las funciones principales del proyecto. Este commit representa
un incremento del producto al final del sprint.

---

### Commit 4
**Mensaje del commit:**


**Metodología:** Scrum  
**Actividad asociada:** Sprint 2 – Desarrollo iterativo  
**Descripción:**  
Se realizaron ajustes y mejoras al contenido del proyecto, manteniendo
commits pequeños y claros para reflejar el avance continuo.

---

### Commit 5
**Mensaje del commit:**


**Metodología:** Scrum + Cascada  
**Actividad asociada:** Pruebas y cierre del proyecto  
**Descripción:**  
Se documentaron las pruebas realizadas y se revisó el proyecto antes del
cierre, asegurando que la documentación estuviera completa.

---

## 3. Beneficios del uso de Git como herramienta de seguimiento

El uso de Git permitió:
- Visualizar el **avance cronológico** del proyecto
- Relacionar cada cambio con una **fase o actividad específica**
- Identificar qué se hizo, cuándo y por qué
- Simular un entorno de trabajo profesional
- Facilitar el control y la revisión del progreso

---

## 4. Conclusión

Git demostró ser una herramienta eficaz no solo para el control de versiones,
sino también para el **monitoreo del progreso del proyecto**. Al utilizar
mensajes de commit claros y alineados con la metodología, fue posible
documentar cada etapa del desarrollo y mantener un seguimiento ordenado
del trabajo realizado.
